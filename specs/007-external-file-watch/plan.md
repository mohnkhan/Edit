## Summary

Feature 007 adds cross-platform filesystem event watching to the editor. When any open buffer's backing file is modified or deleted by an external process, the editor detects the change (via OS-native events: inotify on Linux, kqueue on BSD, FSEvents on macOS) and presents the user with a modal reload-or-keep dialog. Self-writes (editor-initiated saves) are suppressed via a 2-second grace window. Rapid writes are debounced to 1-second bursts. A `--no-watch` flag disables all watching.

## Implementation Phases

### Phase A ‚Äî Watcher Module (`src/watcher/mod.rs`)

New module implementing `FileWatcher`, `WatchEvent`, `WatchEventKind`. Core logic:

1. `FileWatcher::new()` ‚Äî creates `notify::recommended_watcher(tx)` with `std::sync::mpsc::channel`; stores `rx` for drain
2. `watch_path(path)` ‚Äî watches `path.parent()` (directory) with `RecursiveMode::NonRecursive`; ref-counts `watched_dirs`
3. `unwatch_path(path)` ‚Äî decrements refcount; unregisters dir when count reaches zero
4. `try_recv_event(self_write_times)` ‚Äî non-blocking drain:
   - Calls `rx.try_recv()` in a loop
   - For each raw event: filters to known buffer paths (only `Modify` and `Remove` kinds), checks self-write grace window, checks debounce window
   - Returns the first qualifying `WatchEvent`, or `None`

**Constants**:
```rust
const SELF_WRITE_GRACE: Duration = Duration::from_secs(2);
const DEBOUNCE_SECS: Duration = Duration::from_secs(1);
```

**Unit tests** (in `src/watcher/mod.rs #[cfg(test)]`):
- `test_watch_unwatch_refcount` ‚Äî watch same dir twice, assert dir not unwatched until both calls are released
- `test_self_write_suppressed` ‚Äî manually insert recent timestamp in `self_write_times`, assert `try_recv_event` returns `None` for that path
- `test_debounce_coalesces` ‚Äî manually set `last_emitted[path]` to `Instant::now()`, assert second event is suppressed
- `test_unknown_path_ignored` ‚Äî event arrives for path not in any buffer, assert `None` returned

### Phase C ‚Äî App Integration (`src/app.rs`)

New fields added to `App`:
```rust
pub file_watcher: Option<crate::watcher::FileWatcher>,
pub self_write_times: std::collections::HashMap<std::path::PathBuf, std::time::Instant>,
pub pending_external_change: Option<crate::watcher::ExternalChange>,
pub watcher_notice: Option<String>,
```

Changes:
1. `App::new()` ‚Äî if `!config.no_watch`, create `FileWatcher` and call `watch_path` for each initially-opened buffer with a `Some(path)`
2. `App::handle_tick()` ‚Äî after existing auto-save tick: drain `file_watcher.try_recv_event(&self_write_times)`; map to `pending_external_change` (Modified) or `watcher_notice` (Deleted)
3. `Buffer::save()` / `write_to()` call-sites ‚Äî after each successful write to `path`, store `self.self_write_times[path.clone()] = Instant::now()`
4. `App::reload_from_disk(buf_idx)` ‚Äî new method: `Buffer::open(path)` ‚Üí replace `self.buffers[buf_idx]`; re-register watcher for new path if changed
5. `App::handle_action()` ‚Äî add `Action::ReloadFile` (Y/Enter in dialog) and `Action::DismissExternalChange` (N/Esc) handling; clear `pending_external_change`
6. Open/close buffer actions ‚Äî call `watch_path` on open, `unwatch_path` on close

**Unit tests** (in `src/app.rs #[cfg(test)]`):
- `test_no_watch_config_leaves_watcher_none` ‚Äî `config.no_watch=true` ‚Üí `app.file_watcher` is `None`
- `test_self_write_time_recorded_on_save` ‚Äî after `buffer.save()`, assert `self_write_times[path].elapsed() < 1s`
- `test_reload_replaces_buffer_content` ‚Äî set `pending_external_change`, call `handle_action(Action::ReloadFile)`, assert buffer updated
- `test_decline_marks_buffer_dirty` ‚Äî set `pending_external_change`, call `handle_action(Action::DismissExternalChange)`, assert `buffer.dirty == true`
