# Implementation Plan: Soft-Wrap Mode

**Branch**: `005-soft-wrap-mode` | **Date**: 2026-06-19 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/005-soft-wrap-mode/spec.md`

---

## Summary

Add an optional soft-wrap rendering mode that visually folds long lines at the terminal width instead
of scrolling horizontally. The mode is toggled via Alt+Z and a new "Soft Wrap (ext)" View menu item.
Wrap points are computed grapheme-cluster-aware using `unicode-segmentation` + `unicode-width` (both
already present in `Cargo.toml`). A `WrapCache` struct stored in `App` caches break points per
logical line and is invalidated on text edit or terminal resize. All editor operations (Ctrl+S, Find,
undo/redo) remain logically transparent. The setting persists in `$XDG_CONFIG_HOME/edit/config.toml`.

**Project Type**: TUI desktop application (terminal editor)

**Performance Goals**: Wrap cache recompute ≤ 5 ms for a 10 000-line file on typical hardware;
render frame latency unchanged (≤ 50 ms keystroke-to-screen per constitution baseline)

**Constraints**: No new crate dependencies; MSRV 1.74.0; zero changes to buffer/rope/encoding layers

**Scale/Scope**: Single viewport; global toggle; wraps lines up to the maximum rope line length
(tested with lines up to 4 096 chars per research findings)

## Implementation Phases

### Phase A: WrapCache Core (`src/ui/wrap.rs`)

New module. Public API per `contracts/wrap-cache.md`.

**Tasks**:
1. Add `src/ui/wrap.rs` with `WrapCache` struct, `compute()`, `is_stale()`, `visual_to_logical()`, `total_visual_rows()`, `visual_row_count()`.
2. Add `pub mod wrap;` to `src/ui/mod.rs`.
3. Unit tests in `wrap.rs` `#[cfg(test)]`:
   - ASCII-only line: verify break at last whitespace before column limit.
   - No-whitespace line: verify hard-break at column boundary.
   - CJK double-width: verify break before straddling 2-col character.
   - Empty line: `visual_starts[0] == vec![0]`, `visual_row_count == 1`.
   - Line exactly at viewport width: no break (single visual line).
   - Line one char over width: one break → two visual lines.
   - `visual_to_logical` round-trips for all rows.
   - `is_stale()` returns true on width/version change, false otherwise.

### Phase B: Config & Action (`src/config/schema.rs`, `src/input/keymap.rs`)

**Tasks**:
1. Add `soft_wrap: bool` to `Config` with `#[serde(default)]` and `Default::default() = false`.

### Phase C: App State & Dispatch (`src/app.rs`)

**Tasks**:
1. Add `soft_wrap: bool` and `wrap_cache: Option<WrapCache>` fields to `App`.
2. Initialize from `config.soft_wrap` in `App::new()`.
3. Handle `Action::ToggleSoftWrap` in the event dispatch:
   - Check minimum viewport width (≥ 10 cols); show warning if too narrow.
   - Toggle `app.soft_wrap`.
   - Sync `app.config.soft_wrap`.
   - Rebuild/drop `app.wrap_cache`.
   - Reset `buffer.scroll_offset.1 = 0` for all buffers when enabling.
   - Persist config to disk.
4. Invalidate `wrap_cache` on `Event::Resize` when `soft_wrap == true`.
5. Invalidate `wrap_cache` on any buffer edit when `soft_wrap == true` (hook into existing buffer-modified path).

**Tasks**:
1. Add `soft_wrap: bool` and `wrap_starts: &'a [Vec<u32>]` fields to `EditorWidget`.
2. Update `EditorWidget::new()` constructor.
3. In `Widget::render()`: branch on `self.soft_wrap`:
   - `false`: existing code path unchanged.
   - `true`: new wrap rendering path per `contracts/editor-widget.md`:

## Key Invariants to Preserve

1. `buffer.rope` and all encoding operations are untouched — wrap is purely a rendering concern.
2. All `Action` variants other than `ToggleSoftWrap` behave identically regardless of `soft_wrap` state.
3. `WrapCache.visual_starts[L][0] == 0` always holds (first visual sub-line of any logical line starts at byte 0).
4. Wrap break points always land on grapheme-cluster boundaries (enforced in `WrapCache::compute()`).
5. `buffer.scroll_offset.1` (horizontal scroll) is reset to 0 and suppressed while soft-wrap is active.
